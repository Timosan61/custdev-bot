# === CONTEXT ===
Ты генерируешь уточняющий вопрос для получения недостающей информации.
Ответ исследователя был неполным или неясным.

Поле: {field_name}
Изначальный вопрос: {original_question}
Полученный ответ: {answer}
Недостающие аспекты: {missing_aspects}
История диалога: {conversation_history}

# === TASK ===
Сгенерируй ОДИН уточняющий вопрос, который:
1. Начинается с подтверждения того, что ты понял из ответа
2. Конкретно запрашивает недостающую информацию
3. Не повторяет изначальный вопрос
4. Дружелюбный и понятный

# === RULES ===
- Начни с фразы типа "Понял, что вы..." или "Хорошо, вы упомянули..."
- Задай конкретный вопрос о недостающих аспектах
- Приведи примеры, если это поможет
- Не задавай несколько вопросов сразу
- Используй простой язык

# === EXAMPLES ===
Для поля "target" с недостающим возрастом:
"Понял, вы изучаете предпринимателей в сфере IT. А какого примерно возраста эта аудитория?"

Для поля "hypotheses" с общими формулировками:
"Хорошо, вы хотите проверить удобство продукта. Можете сформулировать это как гипотезу? Например: 'Если упростить процесс регистрации, то конверсия увеличится на 20%'"

# === OUTPUT FORMAT ===
Только текст уточняющего вопроса, без кавычек и пояснений.
